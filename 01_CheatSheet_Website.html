Ein System für global eindeutige Kennungen im Web. Die Universal Document Identifier (UDI), später bekannt als Uniform Resource Locator (URL) und Uniform Resource Identifier (URI).

URL

Uniform Resource Locator umgangssprachlich auch Webadresse bezeichnet. Sie ist ein Verweis auf eine Webresource, dies ihren Standort in einem Computernetzwerk hat. Die URL ist eine bestimmte Art der URI.

Anwendung

URL’s treten am häufigsten beim Verweis auf Webseiten (http) auf, aber auch für Dateiübertragung (ftp), Email (Mailto), Datenbankzugriff (JDBC) und viele andere Anwendungen verwendet.

Adressleiste

Die URL wird bei Webbrowsern über der Seiten in einer Adressleiste angezeigt.
Typisches URL Format: http://www.example.com/index.html
http = Scheme
www.example.com = Hostname
index.html = Dateiname

Syntax

Schemen Beispiele Bekannte Beispiele für Schemen sind http, https, ftp, mailto, file, data, irc.
URI-Schematen sollten bei der Internet Assigned Numbers Authority (IANA) regisrtiert werden, obwohl in der Praxis nicht registrierte Systeme verwendet werden.
Syntax
Jede HTTP-URL entspricht der Syntax eines generischen URI. Die generische URI-Syntax besteht aus einer hierarchischen Sequenz von fünf Komponenten:
URI = scheme:[//authority]path[?query][#fragment]
 

URI Aufbau

Die URI umfasst eine nicht leere Schema Komponente (http, https, ftp, etc..), gefolgt von einem Doppelpunkt (:).

Autorisierungskomponente

Nach den zwei Schrägstrichen (//) folgt eine optionale nicht leere Autorisierungskomponente, die Folgendes umfasst:
1. Eine optionale Benutzerinfo-Unterkomponente, die aus einem Benutzernamen und einem optionalen Kennwort bestehen kann(username:password). Die Informationen werden von einem At-Zeichen gefolgt (@). Diese Informationen gelten aber als überholt und werden nicht mehr verwendet aus Sicherheitsgründen.
2. Eine nicht leere Host-Unterkomponente, die entweder aus einem registrierten Namen (einschließlich aber nicht beschränkt auf einen Hostnamen) oder einer IP-Adresse besteht. IPv4-Adressen müssen in Punkt-Dezimal-Notation stehen, und IPv6-Adressen müssen in Klammern ([]) stehen.
3. Eine optionale Port-Subkomponente mit einem Doppelpunkt (:).

Pfadkomponente

Sie besteht aus einer Sequenz von Pfadsegmenten, die durch einen Schrägstrich (/) getrennt sind. Für einen URI wird immer ein Pfad definiert, obwohl der definierte Pfad möglicherweise leer ist (Länge null). Ein Segment kann auch leer sein, was zu zwei aufeinanderfolgenden Schrägstrichen (//) in der Pfadkomponente führt. Eine Pfadkomponente kann genau wie ein Dateisystempfad aussehen oder genau zugeordnet werden, bedeutet jedoch nicht immer eine Beziehung zu einem. Das letzte Segment des Pfades kann als "Slug" bezeichnet werden.

Abfragekomponente

Eine optionale Abfragekomponente, der ein Fragezeichen (?) vorangestellt ist und eine Abfragezeichenfolge nicht hierarchischer Daten enthält.

Fragmentkomponente

Eine optionale Fragmentkomponente, der ein Hash (#) vorangestellt ist. Das Fragment enthält eine Fragmentkennung, die eine Richtung für eine sekundäre Ressource angibt, z. B. eine Abschnittsüberschrift in einem Artikel, der vom Rest der URI identifiziert wird. Wenn die primäre Ressource ein HTML-Dokument ist, ist das Fragment oft ein ID-Attribut eines bestimmten Elements, und Webbrowser scrollen dieses Element in die Ansicht.  

Webserver

Funktion

Die Hauptfunktion eines Webservers ist das Speichern, Verarbeiten und Liefern von Webseiten an Clients. Die Kommunikation zwischen Client und Server erfolgt über das Hypertext Transfer Protocol (HTTP). Am häufigsten werden HTML-Dokumente geliefert, die zusätzlich zum Textinhalt Bilder, Stylesheets und Skripte enthalten können.

Webbrowser

Ein Benutzeragent, üblicherweise ein Webbrowser oder Web-Crawler, initiiert die Kommunikation, indem er eine HTTP-Anforderung für eine bestimmte Ressource anfordert, und der Server antwortet mit dem Inhalt dieser Ressource oder einer Fehlermeldung, falls dies nicht möglich ist. Die Ressource ist normalerweise eine echte Datei auf dem sekundären Speicher des Servers, aber dies ist nicht unbedingt der Fall und hängt davon ab, wie der Webserver implementiert ist. Während die primäre Funktion das Bereitstellen von Inhalten ist, umfasst eine vollständige Implementierung von HTTP auch Möglichkeiten zum Empfangen von Inhalt von Clients. Diese Funktion wird zum Senden von Webformularen, einschließlich des Hochladens von Dateien, verwendet.

ASP

Viele generische Webserver unterstützen auch serverseitige Skripterstellung mit Active Server Pages (ASP), PHP oder anderen Skriptsprachen. Dies bedeutet, dass das Verhalten des Webservers in separaten Dateien geschrieben werden kann, während die eigentliche Server-Software unverändert bleibt. Es wird hauptsächlich zum Abrufen oder Ändern von Informationen aus Datenbanken verwendet.

Webserver andersseitig

Webserver dienen nicht nur zur Bereitstellung des World Wide Web. Sie können auch in Geräten wie Druckern, Routern, Webcams eingebettet sein und nur ein lokales Netzwerk bedienen.  

Webbrowser

Einleitung

Ein Webbrowser (üblicherweise als Browser bezeichnet) ist eine Softwareanwendung zum Zugreifen auf Informationen im World Wide Web. Jede einzelne Webseite, jedes Bild und jedes Video wird durch eine eindeutige URL identifiziert, die es den Browsern ermöglicht, sie auf dem Gerät des Nutzers abzurufen und anzuzeigen.
Ein Webbrowser ist nicht dasselbe wie eine Suchmaschine, aber es wird oft verwechselt. Für einen Nutzer ist eine Suchmaschine nur eine Website wie google.com, die durchsuchbare Daten über andere Websites speichert. Aber um sich auf ihrem Gerät mit Websites zu verbinden und Websites anzuzeigen, muss ein Benutzer einen Webbrowser installiert haben.
Die beliebtesten Webbrowser sind Chrome, Firefox, Safari, Internet Explorer und Edge.

Funktion

Der Zweck eines Webbrowsers besteht darin, Informationsressourcen abzurufen und sie auf dem Gerät eines Benutzers anzuzeigen.
Dieser Prozess beginnt, wenn der Benutzer eine URL wie https://en.wikipedia.org/ in den Browser eingibt.
Praktisch alle URLs im Web beginnen entweder mit http: oder https: was bedeutet, dass der Browser sie mit dem Hypertext Transfer Protocol abruft. Im Falle von https: Die Kommunikation zwischen dem Browser und dem Webserver wird aus Gründen der Sicherheit und Privatsphäre verschlüsselt. Ein anderes URL-Präfix ist „file:“, mit der lokale Dateien angezeigt werden, die bereits auf dem Gerät des Benutzers gespeichert sind.
Sobald eine Webseite abgerufen wurde, zeigt die Rendering-Engine des Browsers sie auf dem Gerät des Benutzers an. Dazu gehören vom Browser unterstützte Bild- und Videoformate.
Webseiten enthalten normalerweise Hyperlinks zu anderen Seiten und Ressourcen. Jeder Link enthält eine URL, und wenn er angeklickt wird, navigiert der Browser zur neuen Ressource. Somit beginnt der Prozess, Inhalte an den Benutzer zu bringen, erneut.
Um das alles zu implementieren, sind moderne Browser eine Kombination aus zahlreichen Softwarekomponenten.
 

Hello World

Die <! DOCTYPE> -Deklaration muss vor dem <html> -Tag die allererste in Ihrem HTML-Dokument sein.
Die Deklaration <! DOCTYPE> ist kein HTML-Tag; Es ist eine Anweisung an den Webbrowser, über welche Version von HTML die Seite geschrieben wird.
Fügen Sie Ihren HTML-Dokumenten immer die Deklaration <! DOCTYPE> hinzu, damit der Browser weiß, welche Art von Dokument zu erwarten ist.

Body Element

Das <body> -Tag definiert den Dokumentkörper.
Das <body> -Element enthält den gesamten Inhalt eines HTML-Dokuments, z. B. Text, Hyperlinks, Bilder, Tabellen, Listen usw.

EM Element

<em>Emphasized text</em>
Das <em> -Tag ist ein Phrasen-Tag. Es render den Text als hervorgehobenen Text.

Head Element

<head> … </head>
Das <head> Element ist ein Container für alle Kopfelemente.
Das <head> -Element kann einen Titel für das Dokument, Skripte, Stile, Metainformationen und mehr enthalten.
Die folgenden Elemente können innerhalb des Elements <head> eingefügt werden:
• <title> (this element is required in an HTML document)
• <style>
• <base>
• <link>
• <meta>
• <script>
• <noscript>

HTML Tag

Das html -Tag teilt dem Browser mit, dass es sich um ein HTML-Dokument handelt.
Es repräsentiert den Stamm eines HTML-Dokuments.
Der Tag ist der Container für alle anderen HTML-Elemente (außer für das -Tag).

Strong Tag

<strong>Strong text</strong>
Das <strong> -Tag ist ein Phrasen-Tag. Es definiert wichtigen Text.

Title Tag

<title>HTML Reference</title>
Das <title> -Tag wird in allen HTML-Dokumenten benötigt und definiert den Titel des Dokuments.
Das <title> Element:
definiert einen Titel in der Browser-Symbolleiste
gibt einen Titel für die Seite an, wenn sie den Favoriten hinzugefügt wird
zeigt einen Titel für die Seite in den Suchergebnissen an
